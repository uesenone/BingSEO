## 🚀 功能概述

必应URL推送工具增强版是一个功能强大的Python GUI应用程序，专为向必应搜索引擎批量提交URL而设计。相比基础版本，增强版提供了更多高级功能和更好的用户体验。

## ✨ 新增功能

### 📊 数据统计与分析
- **提交统计面板** - 显示今日/本周/本月/总计提交次数、成功率
- **URL状态追踪** - 记录每个URL的提交历史和状态
- **图表可视化** - 提交趋势图、成功率统计图
- **导出报告** - 生成Excel/CSV格式的提交报告

### 🔄 灵活提交策略
- **优先级队列** - 为不同URL设置提交优先级（1-10级）
- **灵活间隔** - 根据API响应自动调整提交间隔
- **失败重试** - 自动重试失败的URL提交（可配置重试次数）
- **灵活提交** - 基于优先级和历史记录的灵活提交算法

### 🎨 用户体验优化
- **主题切换** - 支持明暗主题切换
- **多语言支持** - 中英文界面切换
- **快捷键操作** - 常用功能的键盘快捷键
- **系统托盘** - 最小化到系统托盘运行

### 📱 扩展功能
- **URL验证** - 提交前验证URL有效性
- **批量导入** - 支持从Excel、CSV、TXT导入URL
- **定时任务** - 更灵活的定时提交设置
- **多线程优化** - 并发提交提高效率

### 🔧 技术增强
- **SQLite数据库** - 持久化存储提交记录和统计数据
- **缓存机制** - 减少重复请求
- **API限制灵活处理** - 自动处理频率限制
- **错误处理优化** - 更完善的错误处理和日志记录

## 💻 安装要求

### 系统要求
- Windows 10/11
- Python 3.8 或更高版本

### 依赖包安装
```bash
pip install -r requirements_enhanced.txt
```

### 主要依赖包
- `requests` - HTTP请求库
- `pandas` - 数据处理
- `matplotlib` - 图表绘制
- `pystray` - 系统托盘
- `Pillow` - 图像处理

## 🚀 快速开始

### 1. 启动程序
```bash
python bing_url_submitter_enhanced.py
```

### 2. 基本配置
1. 在"主要功能"标签页输入必应API密钥
2. 选择或创建URL文件
3. 点击"保存配置"保存设置

### 3. 开始提交
- **随机提交**: 随机选择一个URL提交
- **批量提交**: 提交所有URL
- **灵活提交**: 基于优先级和历史记录灵活选择URL提交
- **自动提交**: 启用定时自动提交

## 📖 详细功能说明

### 主要功能标签页

#### API密钥配置
- 输入您的必应Webmaster API密钥
- 密钥以星号形式显示保护隐私
- 支持密钥验证和错误提示

#### URL管理
- **文件选择**: 支持选择TXT格式的URL文件
- **URL列表**: 显示URL、优先级、提交次数、成功次数、最后提交时间
- **实时更新**: 自动刷新URL状态信息

#### 提交操作
- **随机提交**: 从URL列表中随机选择一个进行提交
- **批量提交**: 按顺序提交所有URL，支持多线程
- **灵活提交**: 优先提交高优先级和长时间未提交的URL
- **刷新列表**: 重新加载URL文件和状态信息

#### 自动提交设置
- **启用开关**: 控制自动提交功能
- **间隔设置**: 设置自动提交的时间间隔（秒）
- **灵活调度**: 自动选择最适合提交的URL

### 统计分析标签页

#### 统计概览
- **今日统计**: 当天的提交次数、成功次数、成功率
- **本周统计**: 本周的累计统计数据
- **本月统计**: 本月的累计统计数据
- **总计统计**: 历史总计数据

#### 趋势图表
- **提交趋势图**: 显示每日提交次数和成功次数的变化趋势
- **成功率趋势图**: 显示成功率的变化趋势
- **图表导出**: 支持导出PNG、PDF格式的图表

### URL管理标签页

#### URL状态管理
- **状态查看**: 查看每个URL的详细状态信息
- **优先级设置**: 为URL设置1-10级优先级
- **启用/禁用**: 控制URL是否参与提交
- **删除URL**: 从数据库中删除URL记录

#### 批量操作
- **批量设置优先级**: 为多个URL同时设置优先级
- **批量启用/禁用**: 批量控制URL状态
- **数据导出**: 导出URL管理数据

### 设置标签页

#### 基本设置
- **最大重试次数**: 设置失败URL的最大重试次数（默认3次）
- **重试延迟**: 设置重试之间的延迟时间（默认10秒）
- **并发线程数**: 设置并发提交的线程数（默认1个）

#### 界面设置
- **主题选择**: 选择明亮或暗黑主题
- **语言设置**: 选择中文或英文界面
- **系统托盘**: 启用或禁用系统托盘功能

## ⚙️ 配置说明

### 配置文件
程序会自动创建 `bing_config_enhanced.json` 配置文件，包含以下设置：

```json
{
  "api_key": "您的API密钥",
  "auto_submit": false,
  "submit_interval": 60,
  "max_retries": 3,
  "retry_delay": 10,
  "theme": "light",
  "language": "zh",
  "enable_tray": true,
  "concurrent_threads": 1
}
```

### URL文件格式
URL文件应为纯文本格式，每行一个URL：
```
https://example.com
https://example.com/page1
https://example.com/page2
```

## 🗄️ 数据库说明

程序使用SQLite数据库 `bing_submitter.db` 存储数据，包含以下表：

### submissions 表
- 记录每次提交的详细信息
- 包括URL、提交时间、状态码、成功状态、响应数据等

### url_status 表
- 记录每个URL的状态信息
- 包括优先级、总提交次数、成功次数、最后提交时间等

### settings 表
- 存储程序设置信息
- 支持动态配置管理

## 🔧 故障排除

### 常见问题

#### 1. 程序无法启动
- 检查Python版本是否为3.8+
- 确认所有依赖包已正确安装
- 查看错误日志信息

#### 2. API请求失败
- 验证API密钥是否正确
- 检查网络连接
- 确认URL格式正确

#### 3. 数据库错误
- 检查数据库文件权限
- 尝试删除数据库文件重新创建
- 查看详细错误信息

#### 4. 系统托盘不显示
- 确认已安装pystray和Pillow
- 检查系统托盘设置
- 尝试重启程序

### 错误代码说明

| 状态码 | 说明 | 解决方案 |
|--------|------|----------|
| 200 | 提交成功 | 正常状态 |
| 400 | 请求错误 | 检查URL格式和API密钥 |
| 401 | 认证失败 | 验证API密钥 |
| 403 | 权限不足 | 检查API密钥权限 |
| 429 | 请求过于频繁 | 增加提交间隔 |
| 500 | 服务器错误 | 稍后重试 |

### 日志文件
程序运行日志显示在界面中，包含：
- 时间戳
- 操作类型
- 详细信息
- 错误信息

## 📈 性能优化建议

### 提交策略
1. **合理设置间隔**: 避免过于频繁的请求
2. **使用优先级**: 为重要URL设置高优先级
3. **启用重试**: 确保失败的URL能够重新提交
4. **监控成功率**: 定期查看统计数据调整策略

### 系统资源
1. **控制并发数**: 根据系统性能调整线程数
2. **定期清理**: 清理过期的数据库记录
3. **监控内存**: 避免长时间运行导致内存泄漏
